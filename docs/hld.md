# High Level Design (HLD) — COVID-19 Data Platform

Four layers — source, ETL, storage, analytics.

## Layer Summary

| Layer | Tool | Purpose |
|-------|------|---------|
| Source | owid-covid-data.csv | 67 columns · 429,435 rows · country × day |
| Staging load | SSIS Package 1 | CSV → stg_covid_raw (all VARCHAR) |
| Staging validation | SSMS | 7 checks — gate before warehouse load |
| Warehouse build | SSIS Package 2 | dim_date + dim_location (parallel) → fact |
| Test layer | etl_validation table + usp_verify_etl_load | 11 checks — critical pass required |
| Reporting | Power BI / Tableau / SSRS | Star schema — 38 business questions |

## Warehouse Design — Star Schema

```
              dim_date
              (date_id PK)
                   │
                   │ date_id FK
                   │
  dim_location ─── fact_covid_daily
  (location_id PK)  (location_id FK)
```

**Grain:** one row = one country on one date.

**dim_location** — ~195 rows after DQ-01 removes aggregates.
19 columns: static country demographics loaded once.

**dim_date** — ~1,688 rows. Generated by Script Task — not from CSV.
9 columns: date hierarchy for time-based slicing.

**fact_covid_daily** — ~400k rows.
54 columns: all daily measures — cases, deaths, testing,
vaccination, hospitalisation, policy, excess mortality.

## Key Design Principles

- **Dimensions load in parallel:** dim_date and dim_location are
  independent — they load simultaneously. fact_covid_daily waits
  for both via AND constraint before loading.

- **Dimensions load before facts:** FK constraints on fact_covid_daily
  require both dim_location and dim_date to exist first.

- **VARCHAR-first staging:** All 67 columns loaded as strings.
  Type casting happens in Package 2. Prevents package crash
  on one malformed value.

- **Reject table with reason codes:** Bad rows go to dq_rejected_rows
  with DQ-01 through DQ-06 reason codes. Nothing silently dropped.

- **Full reload strategy:** All tables truncated before every load.
  OWID publishes historical corrections — full reload ensures
  warehouse always reflects current state of source file.

- **No ETL-level aggregation:** Data loaded at lowest grain (country × day).
  All weekly, monthly, continental rollups computed at query time.

- **OWID pre-computed fields are pass-through:** Smoothed averages,
  per-million rates, per-hundred percentages already exist in source.
  SSIS casts and loads — does not re-derive.

- **Idempotent:** Safe to re-run. Truncate before every load ensures
  running the package twice produces the same result.
